## 객체지향 쿼리 JPQL 특징

-   테이블이 아닌 엔티티 객체를 조회하는 객체지향 쿼리
-   SQL을 추상화해서 특정 데이터베이스 SQL에 의존하지 않음
-   JPQL은 결국 SQL로 변환됨

## JPQL 기본 문법

-   엔티티 속성은 대소문자를 구분
-   JPQL 키워드는 대소문자를 구분 x
-   클래스명 대신 엔티티명을 사용
-   별칭을 필수로 사용해야 함

## TypeQuery, Query

-   TypeQuery 객체: 반환할 타입을 명확하게 지정할 때 사용
-   Query 객체: 반환 타입을 명확하게 지정할 수 없을 때 사용

## 파라미터 바인딩

-   JPQL은 위치 기준 파라미터, 이름 기준 파라미터 바인딩을 지원
-   위치 기준 파라미터보다는 이름 기준 파라미터 바인딩 방식을 사용하는 것이 더 명확

## 프로젝션

-   SELECT 절에 조회할 대상을 지정하는 것을 의미
-   엔티티 프로젝션: 컬럼 하나하나 나열해서 조회해야하는 SQL과 다르게 원하는 객체 이름만으로 조회 가능
-   임베디드 타입 프로젝션: 임베디드 타입은 조회의 시작접이 될 수 없음
-   스칼라 타입 프로젝션: 숫자, 문자, 날짜와 같은 기본 데이터 타입을 조회
-   여러 값 조회: 프로젝션에 여러 값을 선택하면 TypeQuery를 사용할 수 없고, Query를 사용해야 함
-   new 명령어: SELECT 다음에 new 명령어를 사용하면 반환받을 클래스를 지정할 수 있음(TypeQuery 사용가능)
